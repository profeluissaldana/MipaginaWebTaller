🛠️ Servidor Render y Despliegue Web

Aprenderemos a subir nuestros proyectos de Flask a internet utilizando Render y entenderemos cómo funcionan los servidores modernos.

Render.com (Alojamiento en la nube)
Clase 1

Instalación, configuración e interfaz.

  • ¿Para qué sirve?: Es el puente entre tu código en GitHub y un sitio web en vivo. Compila y actualiza tu aplicación automáticamente cada vez que guardas tus cambios (Push).
  • Ventajas: Es ideal para proyectos escolares en Python (Flask), Node.js o sitios estáticos, y cuenta con un plan gratuito excelente para aprender.
Server Side Rendering (Concepto SSR)
Clase 2

Teoría del renderizado en el servidor.

Es una técnica donde el servidor procesa el código Python/Flask, dibuja la página HTML completa y se la envía lista al navegador del usuario.

  • Diferencia: No envía una pantalla en blanco que se arma con JavaScript, envía el contenido ya masticado y cocinado.
  • Ventajas: Carga más rápido al inicio y ayuda a que Google encuentre tu página más fácilmente (SEO).
🚀 Guía Práctica: Subiendo Flask a Render
Clase 3
Paso 1: Configurar archivos esenciales (.gitignore y dependencias)
  1. Crear el archivo .gitignore: Evita subir archivos innecesarios o carpetas gigantescas (como el entorno virtual). Créalo en la raíz de tu proyecto ejecutando en tu terminal: 
    echo venv/ > .gitignore
    O créalo manualmente en VS Code nombrándolo exactamente .gitignore y añadiendo este contenido básico: 
    venv/
__pycache__/
*.pyc
.env
  2. Crear el archivo de dependencias: Con tu entorno virtual activo, genera el listado de librerías para que Render sepa qué instalar: 
    pip freeze > requirements.txt
    ⚠️ ¡Muy importante! Abre el archivo creado y verifica que la librería gunicorn aparezca en esa lista.
  3. Guarda y sube todos estos cambios preparados a GitHub (git add ., git commit y git push).
Paso 2: Configuración en el panel de Render
  1. Inicia sesión en Render.com usando tu cuenta de GitHub.
  2. Haz clic en New + y selecciona Web Service.
  3. Busca tu repositorio en la lista (puedes usar la lupa de búsqueda si tienes muchos) y presiona el botón azul Connect que aparece al pasar el cursor sobre él.
  4. Completa el formulario de Render con los siguientes datos técnicos:
Campo en Render ¿Qué debemos poner / seleccionar?
Language Python
Build Command pip install -r requirements.txt
Start Command gunicorn app:app
(Si tu archivo principal se llama main.py, usa gunicorn main:app)
Instance Type Seleccionar la opción Free (Gratis).

¡Listo! Haz clic en Create Web Service y espera a que la consola diga Your service is live.

💡 Consejos Técnicos y Seguridad
Clase 4
  • Variables Secretas: Si tu app usa contraseñas o una SECRET_KEY, no las dejes a la vista en GitHub. Agrégalas de forma segura en la sección Advanced -> Environment Variables de Render.
🚨 Solución: ¿Hiciste "git push" ANTES de crear el .gitignore?

Si subiste la carpeta venv/ por accidente, GitHub ya la está rastreando. Para borrarla de la nube sin romper tu computadora, sigue estos pasos en la terminal:

  1. Quitar archivos del seguimiento de Git: Esto le dice a Git que deje de vigilar los archivos del proyecto, respetando el nuevo .gitignore, pero sin borrarlos de tu PC. 
    git rm -r --cached .
  2. Guardar el cambio (Commit): Registramos el estado limpio en nuestro historial local. 
    git add .
git commit -m "Actualizado .gitignore y limpiado archivos rastreados"
  3. Subir la corrección (Push): Enviamos los cambios a GitHub. 
    git push
✓ ¡Listo! La carpeta venv desaparecerá de GitHub pero seguirá intacta en tu computadora.
