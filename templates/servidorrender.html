🛠️ Servidor Render y Despliegue Web

Aprenderemos a subir nuestros proyectos de Flask a internet utilizando Render y entenderemos cómo funcionan los servidores modernos.

Render.com (Alojamiento en la nube)
Clase 1

Instalación, configuración e interfaz.

  • ¿Para qué sirve?: Es el puente entre tu código en GitHub y un sitio web en vivo. Compila y actualiza tu aplicación automáticamente cada vez que guardas tus cambios (Push).
  • Ventajas: Es ideal para proyectos escolares en Python (Flask), Node.js o sitios estáticos, y cuenta con un plan gratuito excelente para aprender.
Server Side Rendering (Concepto SSR)
Clase 2

Teoría del renderizado en el servidor.

Es una técnica donde el servidor procesa el código Python/Flask, dibuja la página HTML completa y se la envía lista al navegador del usuario.

  • Diferencia: No envía una pantalla en blanco que se arma con JavaScript, envía el contenido ya masticado y cocinado.
  • Ventajas: Carga más rápido al inicio y ayuda a que Google encuentre tu página más fácilmente (SEO).
🚀 Guía Práctica: Subiendo Flask a Render
Clase 3
Paso 1: Preparar nuestro repositorio en GitHub
  1. Asegúrate de crear el archivo requirements.txt en tu carpeta del proyecto (con tu entorno virtual activo) ejecutando: 
    pip freeze > requirements.txt
    ⚠️ ¡Muy importante! Verifica que la librería gunicorn aparezca en esa lista.
  2. Sube todos tus cambios a GitHub (git commit y git push).
Paso 2: Configuración en el panel de Render
  1. Inicia sesión en Render.com usando tu cuenta de GitHub.
  2. Haz clic en New + y selecciona Web Service.
  3. Conecta el repositorio de tu proyecto Flask.
  4. Completa el formulario con los siguientes datos técnicos:
Campo en Render ¿Qué debemos poner / seleccionar?
Language Python
Build Command pip install -r requirements.txt
Start Command gunicorn app:app
(Si tu archivo principal se llama main.py, usa gunicorn main:app)
Instance Type Seleccionar la opción Free (Gratis).

¡Listo! Haz clic en Create Web Service y espera a que la consola diga Your service is live.

💡 Consejos Técnicos y Seguridad
Clase 4
  • Variables Secretas: Si tu app usa contraseñas o una SECRET_KEY, no las dejes a la vista en GitHub. Agrégalas de forma segura en la sección Advanced -> Environment Variables de Render.
  • El archivo .gitignore: Recuerda enseñar a tus alumnos a ignorar carpetas pesadas como la del entorno virtual (venv/) antes de subir el código a internet.
😴
¿Tu app tarda en cargar la primera vez? En el plan gratuito, si nadie visita tu página en 15 minutos, el servidor se "duerme". Cuando alguien vuelve a entrar, puede tardar unos 30-50 segundos en despertar. ¡Es completamente normal y no significa que tu código esté mal!
