🧠 Arquitectura de un Proyecto Flask

Cómo se conectan los archivos, carpetas y componentes de un proyecto web profesional.

1. ¿Qué es la arquitectura de un proyecto?

Un proyecto Flask está compuesto por varios archivos y carpetas que trabajan conectados entre sí.

Cada elemento cumple una función específica:

  • Python controla el servidor.
  • HTML estructura las páginas.
  • CSS diseña la interfaz.
  • JavaScript agrega interactividad.
  • Flask conecta todo el sistema.
💡 Entender cómo se relacionan los archivos es una de las partes más importantes del desarrollo web profesional.
Todos los componentes del proyecto trabajan conectados.

2. Estructura de carpetas de Flask

Flask necesita una organización específica de carpetas para funcionar correctamente. 


proyecto_flask/
│
├── app.py
├── requirements.txt
├── .gitignore
│
├── templates/
│   ├── base.html
│   ├── inicio.html
│   ├── frameworkflask.html
│   └── arquitecturaflask.html
│
└── static/
    ├── css/
    │   └── estilos.css
    │
    ├── js/
    │   └── funciones.js
    │
    └── img/
        └── logo.png
⚠️ Flask busca automáticamente:
  • Los HTML dentro de templates.
  • Los archivos estáticos dentro de static.

3. El archivo principal: app.py

El archivo app.py es el cerebro del proyecto.

Allí Flask:

  • Crea el servidor.
  • Define las rutas.
  • Conecta los HTML.
  • Controla la lógica Backend.

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def inicio():
    return render_template('inicio.html')
Código ¿Qué realiza?
Flask(__name__) Crea la aplicación web.
@app.route('/') Define la URL principal.
render_template() Busca HTML en templates.

4. El archivo base.html

El archivo base.html funciona como una plantilla principal reutilizable.

💡 Gracias a esto evitamos repetir el menú, footer y configuraciones en todos los HTML.
📌 Espacio dinámico con block content 

<main>
    {% block content %}
    {% endblock %}
</main>

Ese bloque representa un espacio vacío donde cada página insertará su contenido particular.

5. Herencia de plantillas con extends

Cada página HTML hereda automáticamente la estructura de base.html. 


{% extends 'base.html' %}

{% block content %}

<h1>Contenido de la página</h1>

{% endblock %}
🎉 Gracias a esto todas las páginas mantienen el mismo diseño automáticamente.

6. Navegación con url_for()

Flask utiliza la función url_for() para conectar páginas y rutas. 


href="{{ url_for('inicio') }}"

Flask busca automáticamente la función correspondiente dentro de app.py.

def inicio():

7. Archivos estáticos

Flask utiliza la carpeta static para almacenar CSS, JavaScript e imágenes. 


<link rel="stylesheet"
href="{{ url_for('static', filename='css/estilos.css') }}">


<script src="{{ url_for('static', filename='js/funciones.js') }}">
</script>

8. Entorno virtual (.venv)

Cada proyecto Flask debe tener sus propias dependencias.

📦 Crear entorno virtual 

python -m venv .venv
▶️ Activar entorno virtual 

.venv\Scripts\activate
⛔ Desactivar entorno virtual 

deactivate

9. requirements.txt

Guarda todas las librerías instaladas del proyecto. 


pip freeze > requirements.txt


pip install -r requirements.txt

10. Archivo .gitignore 


.venv/
__pycache__/

11. Publicar en Render

Render permite publicar proyectos Flask gratuitamente. 


PC → GitHub → Render
⚙️ Build Command 

pip install -r requirements.txt
▶️ Start Command 

gunicorn app:app
🎉 Con esto el proyecto Flask quedará publicado en internet.
